# 網路調研工作流

對應 Agent Skill：`web-research`（`~/.claude/skills/web-research/SKILL.md`）。
與 `perplexity-pplx` skill 共用同一份 `.env`：`/Users/tfmacmini4/pplx-cli/.env`。

## 觸發方式

跟 AI 說「幫我調研 <主題>」「網路調研一下 <主題>」「查一下 <主題> 並整理報告」即會啟動。
若只想要快速回答不用存檔，明確說「不用存，快速回答就好」。

## 三階段流程

1. **搜尋**：依 `.env` 的 `WEB_RESEARCH_TOOL_PRIORITY` 順序嘗試工具，湊到 `WEB_RESEARCH_MIN_SOURCES`（預設 3）個不同來源就停止。原始結果存到 `raw/web-research/<時間戳>-<主題>/`。
2. **整理報告**：把所有原始結果彙整成一份結構化 Markdown 報告，存到 `wiki/reports/<時間戳>-<主題>.md`（含引用來源、信心等級）。
3. **回答**：AI 直接在對話中根據報告回答你的原始問題，並附上報告路徑與引用來源。

## 調整工具優先順序 / API Key

編輯 `/Users/tfmacmini4/pplx-cli/.env`：

```dotenv
WEB_RESEARCH_TOOL_PRIORITY=perplexity,tavily,firecrawl,browser
WEB_RESEARCH_MIN_SOURCES=3
WEB_RESEARCH_RAW_DIR=.../raw/web-research
WEB_RESEARCH_REPORT_DIR=.../wiki/reports

PPLX_API_KEY=...
TAVILY_API_KEY=...
FIRECRAWL_API_KEY=...
```

- 改 `WEB_RESEARCH_TOOL_PRIORITY` 的順序即可改變工具優先度，`browser`（Puppeteer 瀏覽器自動化）免 Key，當其他工具都失敗或遇到需要渲染的頁面時才會用到。
- 檢查目前哪些工具可用（不會印出 Key 內容）：

```bash
bash ~/.claude/skills/web-research/scripts/preflight.sh
```
